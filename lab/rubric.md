
# Rubric (50 points)

| Category | Criteria | Points |
|---|---|---:|
| **1. Documentation & Code Quality** | Docstrings added to required functions; clear inline comments explaining callbacks and redraw logic; header with name/date | **10** |
| **2. Slider Control (Event-Driven)** | Slider is present, functional, and updates the plot on user interaction; reasonable sample range; integration with existing controls | **10** |
| **3. Visual Clarity** | Distinct styling for inside vs. outside points; readable legend/title; axes limits/aspect remain appropriate | **5** |
| **4. Reseed Button** | Button present; reseeds RNG; UI shows current seed; plot reliably updates | **10** |
| **5. Accuracy Insight** | On-plot metrics (π estimate, absolute error) displayed and update correctly; brief comment on convergence/accuracy | **10** |
| **6. Reflection** | `REFLECTION.md` (3–6 sentences) addresses event-driven mechanics, your changes, and accuracy vs. sample size | **5** |

**Total: 50 points**

### Scoring Guidance (per category)
- **Full credit**: All requirements met; thoughtful choices; no runtime errors.
- **Partial credit**: Most requirements met; minor issues (styling glitches, small logic bugs).
- **Low/zero**: Missing functionality, incorrect behavior, or crashes.
